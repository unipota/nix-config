# Atuin 履歴同期の手順

Atuin を使用して、複数端末間でシェルのコマンド履歴（history）を同期・共有する手順です。
Nix（Home Manager）の設定で `programs.atuin.settings.auto_sync = true` と同期サーバーの設定を行っている前提です。

## 1. メイン端末（同期元）での初期設定

現在履歴が保存されているメインの端末で、Atuinのアカウントを作成します。
この端末の履歴がサーバーにアップロードされます。

```bash
# アカウントの登録（ユーザー名とメールアドレスは任意）
atuin register -u <あなたのユーザー名> -e <あなたのメールアドレス>

# 登録時に設定したパスワードを入力します
```

登録が完了すると、データの暗号化・復号化に使用される**リカバリーキー（Sync Key）**が生成されます。これを表示して、安全な場所（パスワードマネージャーなど）に必ず保存してください。

```bash
# リカバリーキーの表示
atuin key
```
> [!WARNING]
> このリカバリーキーがないと、他の端末で履歴を復号化（同期）することができません。紛失しないように注意してください。

## 2. 別端末（同期先）での設定

履歴を共有したい別の端末で、先ほどの初期設定で作成したアカウントにログインします。
（Nix設定が既に反映されている必要があります）

```bash
# アカウントにログイン
atuin login -u <あなたのユーザー名>

# パスワードと、先ほど保存したリカバリーキー（Sync Key）を入力します
```

## 3. 同期の確認

ログインが完了すると、自動的に過去の履歴が同期されます。
手動で今すぐ同期させたい場合は、以下のコマンドを実行してください。

```bash
atuin sync
```

これ以降は、Nix で設定した `sync_frequency`（例: 5分）の間隔や、コマンド実行時にバックグラウンドで自動的に同期が行われます。
